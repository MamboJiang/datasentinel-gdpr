# Contributing

## Language

Use English for documentation, issues, pull requests, commit messages, comments, and user-facing text.

## Change Discipline

- Keep changes small, testable, and reversible.
- Do not add implementation code before the relevant acceptance criteria are approved.
- Do not introduce dependencies, external services, or abstraction layers for unapproved future needs.
- Update the relevant document whenever requirements, behavior, interfaces, states, or acceptance criteria change.

## Pull Requests

Each pull request should include:

- The goal of the change.
- The affected files and expected impact.
- The validation command or review method used.
- Any remaining risks or follow-up work.
